## 109.1 Fundamentals of internet protocols

### Key Areas

- Understand IPv4/IPv6 addressing, subnetting, and routing basics.
- Understand TCP/UDP roles and common service ports.
- Recognize protocol layering and packet flow concepts.

### Key Commands

```format
ip
ss
netstat
route
```

### Key Paths

```format
/etc/protocols
/etc/services
```

### Summary

This objective validates conceptual networking knowledge required for practical Linux administration and troubleshooting.

### Possible Exam Questions

1. What is the difference between TCP and UDP?
2. Which file maps service names to default ports?
3. What does CIDR notation `/24` represent?
4. Why can IPv6 and IPv4 coexist on a host?

## 109.2 Basic network configuration

### Key Areas

- Configure interface addresses, routes, and hostnames.
- Bring interfaces up/down and validate link state.
- Understand static vs dynamic configuration approaches.

### Key Commands

```format
ip addr
ip link
ip route
ifconfig
route
hostname
hostnamectl
dhclient
```

### Key Paths

```format
/etc/hostname
/etc/hosts
/etc/network/interfaces
/etc/sysconfig/network-scripts/
/etc/NetworkManager/
```

### Summary

Expect hands-on tasks: assigning addresses, adding routes, and ensuring settings persist according to distro tooling.

### Possible Exam Questions

1. How do you add a temporary IPv4 address with `ip`?
2. Where is static interface config stored on Debian-style systems?
3. Which command displays the kernel routing table?
4. How do you set a persistent hostname on systemd systems?

## 109.3 Basic network troubleshooting

### Key Areas

- Test connectivity, path, and name resolution.
- Diagnose local listening sockets and remote reachability.
- Identify firewall/routing/interface issues from command output.

### Key Commands

```format
ping
ping6
traceroute
tracepath
mtr
ss
nc
host
dig
```

### Key Paths

```format
/proc/net/
```

### Summary

High exam weight: you should rapidly move from symptom to cause using layered troubleshooting commands.

### Possible Exam Questions

1. Which command shows open TCP/UDP listening sockets?
2. How does `traceroute` help isolate network failures?
3. What does a successful ping prove, and what does it not prove?
4. Which tools query DNS records directly?

## 109.4 Configure client side DNS

### Key Areas

- Configure resolver behavior and name server order.
- Understand resolver search domains and query sequence.
- Verify effective DNS settings and troubleshoot lookup issues.

### Key Commands

```format
resolvectl
host
dig
nslookup
getent
```

### Key Paths

```format
/etc/resolv.conf
/etc/hosts
/etc/nsswitch.conf
/etc/systemd/resolved.conf
```

### Summary

Know how Linux resolves names through local files and DNS, and how resolver configuration interacts with modern services like `systemd-resolved`.

### Possible Exam Questions

1. Which file defines host lookup order between files and DNS?
2. What is the purpose of the `search` directive in resolver config?
3. Why might `/etc/resolv.conf` be autogenerated?
4. Which command can test NSS-based name resolution behavior?
